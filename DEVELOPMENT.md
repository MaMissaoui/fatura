# Development Guide

Fatura is a cross-platform desktop invoicing app built with [Wails v2](https://wails.io) (Go backend) and React (frontend). This guide covers everything you need to run the app locally and produce release builds.

---

## Prerequisites

| Tool | Minimum version | Install |
|------|----------------|---------|
| [Go](https://go.dev/dl/) | 1.21 | `brew install go` |
| [Node.js](https://nodejs.org) | 18 LTS | `brew install node` |
| [pnpm](https://pnpm.io) | 9+ | `npm install -g pnpm` |
| [Wails CLI](https://wails.io/docs/gettingstarted/installation) | v2 | `go install github.com/wailsapp/wails/v2/cmd/wails@latest` |

### Linux only — additional system packages

```bash
sudo apt-get install libgtk-3-dev libwebkit2gtk-4.1-dev \
  libayatana-appindicator3-dev librsvg2-dev
```

Make sure `$(go env GOPATH)/bin` is on your `PATH` so the `wails` command is found:

```bash
export PATH="$PATH:$(go env GOPATH)/bin"
```

Verify everything is ready:

```bash
wails doctor
```

## First-time setup

```bash
# Clone the repository
git clone https://github.com/MaMissaoui/fatura.git
cd fatura

# Install Go dependencies
go mod download

# Install frontend dependencies
pnpm install
```

## Running in development

```bash
wails dev
```

This single command:
1. Starts the Go backend (compiles and launches the app window)
2. Starts the Vite dev server on `http://localhost:5173`
3. Enables hot-reload for frontend changes
4. Regenerates the TypeScript bindings in `wailsjs/` from your Go code

The app window opens automatically. Frontend edits refresh instantly; Go code changes trigger a backend recompile.

### Frontend-only development (no app window)

If you only need to work on UI without running the full desktop app:

```bash
pnpm dev
```

Opens the Vite dev server at `http://localhost:5173` in a browser. Backend calls will fail (no Go process), but layout and component work is fully functional.

## Building for production

### Current platform

```bash
wails build
```

Output is placed in `build/bin/`.

### Cross-platform targets

```bash
# macOS — Apple Silicon
wails build -platform darwin/arm64

# macOS — Intel
wails build -platform darwin/amd64

# Linux
wails build -platform linux/amd64

# Windows
wails build -platform windows/amd64
```

### macOS universal binary (arm64 + amd64)

```bash
wails build -platform darwin/universal
```

## Internationalization

Translation strings are extracted from the source with:

```bash
pnpm extract
```

Translation files live in `src/locales/` as `.po` files. Add new translations there and re-run the app.

---

## Database

The SQLite database is stored at:

| Platform | Path |
|----------|------|
| macOS | `~/Library/Application Support/Fatura/sqlite.db` |
| Linux | `~/.config/Fatura/sqlite.db` |
| Windows | `%APPDATA%\Fatura\sqlite.db` |

Migrations in `db/migrations/` run automatically on startup. No manual steps are required.

### Backup & restore

Use **Settings → Backup** inside the app to create or restore a database backup via a file dialog.

## Troubleshooting

**`wails: command not found`** — Add `$(go env GOPATH)/bin` to your `PATH` (see Prerequisites above).

**Stale Wails bindings** — If the frontend can't find a Go method, regenerate the bindings:
```bash
wails dev   # or: wails generate module
```

**Port 5173 already in use** — Stop any running Vite instances before launching `wails dev`.

**SQLite migration error on first run** — Delete the database file and restart to let migrations run fresh on a clean database.
